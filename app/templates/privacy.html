Privacy & Data Handling

ELORA is a local prototype designed for a final year project. It focuses on modelling obesity risk and visualising lifestyle trends, while keeping data on the user's own device.

What data does ELORA store?

  • Daily records: date, age, gender, height, weight
  • Simple lifestyle fields: family history flag, activity level, water intake
  • Derived BMI value for each record

No name, email address, student ID, home address or contact details are stored.

Where is the data stored?

  • Data is stored in a local SQLite database file under the ELORA project folder.
  • The prototype does not send records to any remote server or cloud service.

How can the user control their data?

  • Use the Download history as CSV option on the History page to export a copy.
  • Use the Clear history button to permanently delete all records from the local database.

Use in research / teaching context

This prototype is intended for demonstration and educational purposes only. It does not provide medical diagnosis or treatment recommendations. Any health-related decisions should be made with a qualified clinician.
